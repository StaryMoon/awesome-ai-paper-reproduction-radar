# Daily Radar Note - 2026-06-22

Today upgrades the radar from prose-only code-status wording to explicit dataset fields.

## Schema Update

Each paper entry in `data/reproductions.json` now includes:

```json
{
  "official_code_url": "",
  "official_code_status": "not_indexed",
  "starter_type": "unofficial"
}
```

The dataset metadata now records:

- `schema_version: 0.2.0`
- `code_status_fields: ["official_code_url", "official_code_status", "starter_type"]`

## Why This Helps

The radar is easier to reuse when code availability is stored as data instead of being implied by README prose. It also makes future filtering possible:

- show entries with official code indexed;
- show entries that still need official-code review;
- separate unofficial starters from official implementations;
- build a cleaner code-status dashboard later.

## Generated Output

The README repository tables and static HTML page now expose an explicit `Official code status` / `Official Code` column. For now, all current starter entries default to `official_code_status: not_indexed`, which means this radar has not indexed an official-code URL yet.
